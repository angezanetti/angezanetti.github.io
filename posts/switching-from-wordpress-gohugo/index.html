Switching From Wordpress to a static website (hugo)

I’m a big fan of Wordpress, I start using it almost 10 years ago, I also created/adapt a ton of plugins/themes during the first part of my career. It’s a wonderful tool to create a website in a few minutes, the backoffice is really easy for the non-techy and so on. But, it’s also a pain in the ___ to maintain, you have to have a SQL database, to update the plugins on a regular basis, you need to check the spam folder etc…

Medium?

I wanted something lighter

I think you get my point, I’m tired of the all actions needed to maintain a WP blog. Even a server, I just want a place somewhere to publish some (too) occasionnal blog posts. No database, no apache, no plugins, no comments (i.e. no spam), no cache, no PHP.

HTML + CSS + content. That’s it At this time I didn’t had a strong opinion on the engine, Jekyll, Hugo, Pelican, whatever langage, as soon as I have clean HTML at the exit!

I feared the migration

Also, I don’t know if you ever tried but, moving a WP blog from a server to another is so much pain. You need to dig into the db, change the url. 1 time out of 2 you lost a day on a migration. So, I was ok to migrate to a cooler/lighter blog engine but I definitely don’t have the time to spend a day on crashing my blog.

And I discovered the doc of gohugo.io and it’s plugin WordPress to Hugo Exporter. Look at the description:  One-click WordPress plugin that converts all posts, pages, taxonomies, metadata, and settings to Markdown and YAML which can be dropped into Hugo.

One click? Really?

I gave it a try, and yes, it’s insane but it works in 2 lines.

php hugo-export-cli.php
wp hugo-export > export.zip

And BOOM all your posts can be dropped to Hugo, then copy/paste them into the content section, launch your local server with hugo server and VOILA.

Hosting + Design

For the design 2 solutions: you take a theme in the theme library on the hugo doc’s website. Or you build your own. I choose the hermit theme, tweak it a bit - I hope you like the shiny pink color :O

For the hosting you can it’s easy because it’s HTML + CSS after all. So you can drop it pretty much wherever you want. I choose Github Pages, because I’m a fan of Github and you need to version your code after all! Once again, the doc is pretty well written, so just follow it and you should be good!
